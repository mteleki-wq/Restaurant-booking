Complete setup guide

Restaurant booking system
— from zero to taking bookings

Everything on one page: the database script, the settings to change, how to put it online, and how to run it on service nights. Total setup time: about 15 minutes.

1

Create a free Supabase project

  1. Go to supabase.com and sign up (free plan is plenty).
  2. Tap New project. Give it a name like restaurant-bookings, choose a database password (save it somewhere safe — you rarely need it again), pick the region closest to you (Sydney), and create.
  3. Wait a minute or two while the project builds.

Already have a Supabase project (e.g. for ChefLink)? You can use it — this script won't touch your existing tables.

2

Run the database script

  1. In Supabase, open SQL Editor (left menu) → New query.
  2. Tap Copy SQL below, paste it in, and press Run.
-- ============================================================
-- RESTAURANT BOOKING SYSTEM — complete Supabase setup
-- (one script: tables, security, functions, live updates)
-- Safe to run more than once.
-- ============================================================

-- 1. SETTINGS (one row holding capacity numbers)
create table if not exists public.booking_settings (
  id int primary key default 1,
  total_capacity int not null default 100,
  online_seats  int not null default 80,
  constraint single_row check (id = 1)
);

insert into public.booking_settings (id, total_capacity, online_seats)
values (1, 100, 80)
on conflict (id) do nothing;

-- 2. BOOKINGS
create table if not exists public.bookings (
  id           uuid primary key default gen_random_uuid(),
  ref          text not null unique,
  booking_date date not null,
  slot_id      text not null,
  slot_label   text not null,
  arrival_time text,
  guests       int  not null check (guests between 1 and 50),
  name         text not null,
  phone        text not null,
  email        text not null,
  notes        text,
  status       text not null default 'confirmed'
               check (status in ('confirmed','cancelled')),
  created_at   timestamptz not null default now()
);

-- if you ran an older version without arrival_time, this adds it
alter table public.bookings add column if not exists arrival_time text;

create index if not exists bookings_date_slot
  on public.bookings (booking_date, slot_id)
  where status = 'confirmed';

-- 3. SECURITY (row level security)
alter table public.bookings enable row level security;
alter table public.booking_settings enable row level security;

-- staff (signed-in users) can manage bookings and settings
drop policy if exists "staff read bookings"   on public.bookings;
drop policy if exists "staff update bookings" on public.bookings;
drop policy if exists "staff delete bookings" on public.bookings;
create policy "staff read bookings"   on public.bookings for select to authenticated using (true);
create policy "staff update bookings" on public.bookings for update to authenticated using (true);
create policy "staff delete bookings" on public.bookings for delete to authenticated using (true);

drop policy if exists "staff read settings"   on public.booking_settings;
drop policy if exists "staff update settings" on public.booking_settings;
create policy "staff read settings"   on public.booking_settings for select to authenticated using (true);
create policy "staff update settings" on public.booking_settings for update to authenticated using (true);

-- the public gets NO direct table access — only the two
-- functions below. Customer details stay private.

-- 4. AVAILABILITY (safe for the public: numbers only)
create or replace function public.get_availability(p_date date)
returns json
language plpgsql security definer set search_path = public
as $$
declare
  v_total int;
  v_seats int;
  v_taken json;
begin
  select total_capacity, online_seats into v_total, v_seats
    from public.booking_settings where id = 1;

  select coalesce(json_object_agg(slot_id, s), '{}'::json) into v_taken
  from (
    select slot_id, sum(guests) as s
    from public.bookings
    where booking_date = p_date and status = 'confirmed'
    group by slot_id
  ) t;

  return json_build_object(
    'total_capacity', v_total,
    'online_seats',   v_seats,
    'taken',          v_taken
  );
end;
$$;

grant execute on function public.get_availability(date) to anon, authenticated;

-- 5. CREATE BOOKING (atomic — overbooking is impossible)
-- remove any older version without the arrival time
drop function if exists public.create_booking(date, text, text, int, text, text, text, text);

create or replace function public.create_booking(
  p_date date,
  p_slot_id text,
  p_slot_label text,
  p_arrival text,
  p_guests int,
  p_name text,
  p_phone text,
  p_email text,
  p_notes text
)
returns json
language plpgsql security definer set search_path = public
as $$
declare
  v_seats int;
  v_taken int;
  v_ref   text;
begin
  if p_guests is null or p_guests < 1 or p_guests > 50
     or coalesce(trim(p_name),'')  = ''
     or coalesce(trim(p_phone),'') = ''
     or p_email is null or position('@' in p_email) = 0 then
    return json_build_object('ok', false, 'reason', 'invalid');
  end if;

  -- one booking at a time per date + sitting (no race conditions)
  perform pg_advisory_xact_lock(hashtext(p_date::text || '|' || p_slot_id));

  select online_seats into v_seats from public.booking_settings where id = 1;

  select coalesce(sum(guests), 0) into v_taken
  from public.bookings
  where booking_date = p_date and slot_id = p_slot_id and status = 'confirmed';

  if v_taken + p_guests > v_seats then
    return json_build_object('ok', false, 'reason', 'full',
                             'seats_left', greatest(v_seats - v_taken, 0));
  end if;

  v_ref := 'BK-' || to_char(now(), 'YYYYMMDD') || '-'
        || upper(substr(md5(random()::text || clock_timestamp()::text), 1, 5));

  insert into public.bookings
    (ref, booking_date, slot_id, slot_label, arrival_time, guests, name, phone, email, notes)
  values
    (v_ref, p_date, p_slot_id, p_slot_label, nullif(trim(p_arrival), ''), p_guests,
     trim(p_name), trim(p_phone), lower(trim(p_email)), nullif(trim(p_notes), ''));

  return json_build_object('ok', true, 'ref', v_ref);
end;
$$;

grant execute on function public.create_booking(date, text, text, text, int, text, text, text, text)
  to anon, authenticated;

-- 6. LIVE UPDATES for the staff panel (green "Live" dot)
do $$
begin
  if not exists (
    select 1 from pg_publication_tables
    where pubname = 'supabase_realtime'
      and schemaname = 'public'
      and tablename = 'bookings'
  ) then
    alter publication supabase_realtime add table public.bookings;
  end if;
end $$;

-- ============================================================
-- DONE. Continue with Step 3 of the guide (create staff login).
-- ============================================================

This one script creates the tables, locks down privacy (customers can never see other customers' details), installs the overbooking protection, and switches on live updates for the staff panel. It's safe to run again later if you're ever unsure.

3

Create the staff login

  1. In Supabase: Authentication → Users → Add user → Create new user.
  2. Enter the email and password your staff will use, and tick Auto Confirm User.

You can add more staff users the same way any time. Anyone with a login can view, cancel and delete bookings and adjust capacity.

4

Copy your two connection values

  1. In Supabase: Project Settings (gear icon) → API.
  2. Copy the Project URL (looks like https://abcdefgh.supabase.co).
  3. Copy the anon public key (the long one under "Project API keys").
Only ever use the anon public key in the booking page. Never the service_role key — that one bypasses all security and must stay secret.
5

Edit the CONFIG in the booking page

Open restaurant-booking-supabase.html in your editor and find the section marked CONFIG near the top of the JavaScript. Change:

SettingWhat to put
supabaseUrlYour Project URL from Step 4
supabaseAnonKeyYour anon public key from Step 4
restaurantNameThe restaurant's name (appears everywhere)
contactPhone / contactEmailReal contact details (shown when fully booked)
timeSlotsYour sittings — each has a label and an anchor time. Default: 6:00 PM and 8:00 PM.
serviceStart / serviceEndFirst and last bookable arrival times (24h format, e.g. "17:30" and "21:30")
timeStepMinutesGap between pickable times (30 by default)
openDaysDays you take bookings. 0=Sun … 6=Sat. Closed Mondays: [0,2,3,4,5,6]
minBookingSize / maxBookingSizeSmallest / largest group bookable online
brandColor / brandColorDarkYour colours (any hex codes)

Capacity (total seats and online seats) is NOT in the file — it lives in the database and staff can change it from the panel. It starts at 100 total / 80 online.

6

Put the page online

Any static hosting works. The GitHub Pages route (free):

  1. Upload restaurant-booking-supabase.html to your GitHub repository.
  2. Open the file's page address, e.g. yourname.github.io/bookings.html.
  3. To embed in an existing website: link to it, or open it in a popup — the built-in Book a Table button shows the modal pattern; you can also load the page in an <iframe>.
7

Test it (5 minutes)

  • Open the page — the sitting buttons should show "80 seats left".
  • Make a booking with your own details → you should get a reference like BK-20260706-4X8Q.
  • Tap Staff in the footer → sign in with the Step 3 user.
  • Your booking is in the table, capacity cards updated, and a green Live dot shows.
  • On a second device, make another booking → it should appear on the staff screen within a second, flash green, and (if the 🔔 box is ticked) chime.
  • Cancel it → seats return instantly on both screens.
  • Tap Export CSV → the day's list downloads.
8

Optional: email confirmations (n8n)

The system is complete without this — it only adds automatic emails.

  1. In n8n: Import from File → booking-email-workflow.json → select your Gmail credential in the three Gmail nodes → set the restaurant's email in the alert node → Activate → copy the webhook Production URL.
  2. In Supabase: Database → Webhooks → create one on table bookings, events Insert + Update, method POST, paste the n8n URL.
  3. To show the exact arrival time in emails, use arrival_time from the record in the Gmail templates.

10

If something's not working

SymptomFix
"This booking page isn't connected yet"The Supabase URL or key in CONFIG is still the placeholder — paste your real values (Step 4/5).
"We couldn't reach the booking system"Check the internet connection; check the URL has no typo and starts with https://.
Booking fails after confirmingThe SQL script hasn't been run (or an old version was). Re-run the Step 2 script — it's safe to re-run.
Staff sign-in failsCheck the user exists in Authentication → Users and was auto-confirmed. Passwords are case-sensitive.
No green Live dotRe-run the Step 2 script (it enables live updates). The Refresh button and auto-refresh still work meanwhile.
No chime on iPadUntick and re-tick the 🔔 box — iPad only allows sound after you've tapped something.
Seats look wrongCancelled bookings don't count. Check the staff panel capacity numbers; remember capacity is per sitting, per day.
